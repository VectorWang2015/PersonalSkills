# 信用减值损失调节利润的识别方法

## R — 原文 (Reading)

> 信用减值损失，主要是银行用以应对未来风险的一种方法，同时也是银行调节利润波动的一种有效手段。银行的生意特点是利润前置而风险后置。
>
> — 价投谷子地，《看透银行》第5章 5.1节

> 通常，银行在营收增长比较多的时候会倾向于多计提减值损失，而在营收不佳的时候少计提减值损失，从而达到平滑利润的目的。
>
> — 价投谷子地，《看透银行》第5章 5.1节

> 有了信用减值损失这一利润调节工具，投资者所看到的银行会计报表中的净利润和净利润增速就是经过银行调节后的。也就是说，两家银行的净利润增速相似，并不代表两家银行的实际经营情况相近。
>
> — 价投谷子地，《看透银行》第5章 5.1节

## I — 方法论骨架 (Interpretation)

信用减值损失是银行损益表中最易受人为操纵的分项，其本质是连接"今天收到的利息"和"未来可能发生的损失"之间的蓄水阀门。银行通过调节阀门开度，在不同年份间腾挪利润。

**核心方法论分三层递进：**

**第一层：还原真实利润。** 不看净利润，看"拨备前利润"（税前利润 + 信用减值损失）。净利润是经过阀门调节后的结果，拨备前利润才是银行主营业务的真实产出。

**第二层：判断调节方向。** 对比拨备前利润增速与净利润增速的差距：
- 拨备前利润增速 **>** 净利润增速 → 银行在**蓄水**（多提减值，隐藏当期利润为未来储备）
- 拨备前利润增速 **<** 净利润增速 → 银行在**放水**（少提减值，以减值准备池的存量水填补当期利润缺口）

**第三层：识别经营意图（"三种牌桌"框架）。** 站在银行行长视角反推动机：

| 场景 | 营收状态 | 不良状态 | 银行行为 | 意图判断 |
|------|---------|---------|---------|---------|
| 一手好牌 | 营收好 | 不良可控 | 加大计提、主动暴露不良、加大核销 | 利好——未雨绸缪，为未来储备安全垫 |
| 好坏参半 | 营收差/不良高 | 尚可维持 | 减缓不良暴露、将不良藏于关注类 | 警惕——在平滑报表，需观察是否恶化 |
| 一手烂牌 | 营收差 | 不良高 | 全部隐藏、不良偏离度失控 | 危险——粉饰不可持续，迟早暴雷 |

三层递进关系：第一层给出真实数据 → 第二层判断调节方向 → 第三层定性银行意图，最终形成完整的利润质量判断。

## A1 — 书中的应用 (Past Application)

### 案例：银行Z与银行X的利润增速对比（2017年）

- **问题**：表面看两家银行2017年净利润增速分别为13.24%和6.27%，相差约7个百分点，似乎差距不大。投资者如果只看净利润增速，很可能得出两家经营质量相近的错误结论。
- **方法论的使用**：作者计算了两家银行的拨备前税前利润增速，发现银行Z是正增长，银行X却是负增长，两者增速差距高达约17个百分点——远大于净利润增速的7个百分点差距。进一步追踪发现，银行X在2017年大幅降低了信用减值损失计提力度（同比减少30.75%）。
- **结论**：银行X是在通过降低减值损失来消除拨备前利润增速负增长的影响。其真实经营状况（拨备前利润负增长）远差于净利润增速所呈现的面貌。
- **结果**：仅看净利润增速会被银行调节利润的手段完全迷惑。必须比较拨备前利润增速，才能还原真实经营趋势。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. **横向对比银行时**：用户列出2家以上银行的净利润增速，想判断哪家更好——需要先还原拨备前利润再比较。
2. **质疑某银行利润真实性时**：用户看到某银行净利润增速与行业背离、或营收下滑却净利润正增长，怀疑有"猫腻"。
3. **读银行年报做排雷**：用户在分析目标银行财报，想判断利润含金量。
4. **预判银行未来暴雷风险时**：用户怀疑某银行在用拨备"蓄水池"腾挪利润，想预判这种模式还能撑多久。

### 语言信号（用户的话里出现这些就应激活）

- "XX银行的净利润增速和YY银行差不多，哪家更好？"
- "这家银行营收都负增长了，净利润居然是正的，怎么回事？"
- "帮我看看这家银行的利润有没有水分"
- "比较一下这几家银行的盈利质量"
- "银行利润增速这个数字可信吗？"

### 与相邻 skill 的区分

- 与净息差分析的区别：净息差分析关注盈利的来源结构（负债端优势 vs 资产端冒险），本 skill 关注盈利的数字真实性（有没有被减值调节粉饰）。
- 与不良贷款生命周期分析的区别：不良生命周期分析资产的恶化过程，本 skill 分析银行为应对不良而做的利润腾挪——前者看风险本身，后者看银行怎么处理风险对利润的影响。
- 与 RORWA 评估的区别：RORWA 是效率指标（单位风险资产产出多少利润），本 skill 是质量指标（报表上的利润是否可信）。

## E — 可执行步骤 (Execution)

当 skill 被激活后，按以下步骤执行：

1. **计算拨备前利润**
   - 公式：拨备前利润 = 税前利润 + 信用减值损失
   - 数据来源：银行损益表中的"税前利润"和"信用减值损失"（或"资产减值损失"）
   - 完成标准：获得至少最近2-3年的拨备前利润值（单年无对比意义）

2. **计算并对比增速差距**
   - 计算拨备前利润同比增速（建议2-3年复合增速，避免单年波动误导）
   - 计算净利润同比增速（同样的时间跨度）
   - 对比：增速差距 = 拨备前利润增速 - 净利润增速
   - 判停条件：若两者增速差距在±3个百分点以内 → 银行未显著调节，跳到步骤4；若差距>3个百分点 → 进入步骤3

3. **"三种牌桌"场景识别**
   - 收集辅助数据：营收增速、不良率趋势、不良偏离度（逾期90+/不良）、拨备覆盖率变化趋势
   - 将当前数据与下表对照：
     - 拨备前利润增速 **显著>** 净利润增速，且营收正增长、不良可控 → **一手好牌**（蓄水，利好）
     - 拨备前利润增速 **显著<** 净利润增速，且营收/不良有一项压力 → **好坏参半**（放水，警惕，需追踪后续季度恶化方向）
     - 拨备前利润增速 **显著<** 净利润增速，且营收负增长、不良偏离度>100% → **一手烂牌**（粉饰不可持续，高危险信号）
   - 完成标准：明确判定银行处于哪种牌局，并给出判断依据

4. **形成综合判断**
   - 若为一手好牌 → 利润质量高，真实经营好于报表呈现，未来有释放利润空间
   - 若为好坏参半 → 利润质量存疑，需持续观察，建议结合新生成不良率判断恶化概率
   - 若为一手烂牌 → 利润质量极差，存在暴雷风险，回避或做空
    - 完成标准：给出明确的投资建议方向（优势/中性/回避）

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 分析的是非银行企业（普通企业的减值损失调节空间远小于银行，方法论不通用）
- 需要分析的银行未披露"信用减值损失"科目明细（数据不可得则无从计算拨备前利润）
- 用户关心的核心问题是银行的负债成本或资产结构问题（这不属于利润质量范畴，应走净息差拆解或负债成本评估 skill）
- 银行处于极端宏观冲击期（如系统性金融危机），此时所有银行的减值逻辑都可能被打乱，"三种牌桌"框架失效

### 作者在书中警告的失败模式

**只看净利润增速，忽略拨备前利润（ce05）：** 这是最典型的投资者误区。银行是利润前置、风险后置的行业，净利润增速是被信用减值损失"化妆"后的数据。营收负增长但通过少提减值维持净利润正增长（如兴业银行2017年营收-10.89%而净利润+6.22%），本质是透支未来安全垫。直接用净利润增长率比较银行，等于拿化妆后的照片比美丑。

### 作者的盲点 / 时代局限

- **招行重仓偏见**：作者是招商银行的长期重仓持有者（自述持仓超10年、6年10倍），全书以招行为"最优解"框架。这可能导致对"高拨备、严格计提"模式的银行给予过高偏好，而忽略了某些银行（如邮储、宁波银行等）以不同路径实现的合理拨备水平。并非拨备越厚越好——过犹不及的超额拨备可能意味着管理层缺乏有效的资本配置能力。
- **巴塞尔III终局未覆盖**：本书出版于2021年，TLAC（总损失吸收能力）等2024-2025年生效的新监管要求未被纳入分析框架。新规下银行的减值计提策略可能发生结构性变化。
- **拨备前利润也有盲区**：拨备前利润排除了减值损失的主观干扰，但它仍包含"公允价值变动损益"等非现金科目和一次性项目。质量最高的分析应在拨备前利润基础上进一步剔除一次性非经常性损益。

### 容易混淆的邻近方法论

- 普通企业的"扣非净利润"分析：非金融企业的核心利润调节手段是折旧政策、收入确认时点和资产减值回拨，与银行的信用减值损失逻辑完全不同，不可混用。
