# 不良贷款生命周期 — 四池三过程 + 三维度交叉验证

## R — 原文 (Reading)

> 整个不良贷款的产生有4个"池子"和3个主要的过程。4个池子分别是池子1：全部贷款；池子2：不良贷款；池子3：贷款减值准备；池子4：减值前利润。3个主要过程是过程1：新生成不良贷款；过程2：计提资产减值损失；过程3：核销不良贷款。
>
> — 那一水的鱼，第6章 6.2.4 不良贷款的生命周期

> 最容易被人忽略、但又最重要的指标是新生成不良率。新生成不良率=新生成不良贷款÷贷款总额。这个指标反映了在一段时间内有多少比例的贷款从池子1流向池子2，这是最重要的原生指标。而不良率实际上是经过核销、催收等手段修饰过的余额比率，其中很多信息有所缺失。
>
> — 那一水的鱼，第6章 6.2.4

> 不良率这个数据不是原生数据，而是经过银行修饰后的数据……想要评判不同银行之间的资产质量水平，还需要依赖其他指标。衡量一家银行风险管理水平的核心指标包括：风险水平、风险迁徙率和风险抵补。
>
> — 那一水的鱼，第6章 6.2.5 / 第10章 10.3.2

---

## I — 方法论骨架 (Interpretation)

银行的不良贷款不是一个静态余额，而是一个**有进水口和出水口的动态水池系统**。

**四池三过程模型**将资产质量的演变还原为四个蓄水池和三条管道：

| 池子 | 含义 | 角色 |
|------|------|------|
| 池子1：全部贷款 | 所有放出去的贷款 | 来源 |
| 池子2：不良贷款 | 已被认定为不良的贷款 | 问题蓄积处 |
| 池子3：贷款减值准备 | 从利润中计提的风险准备金 | 安全垫 |
| 池子4：减值前利润 | 计提前的营业利润 | 资金来源 |

| 过程 | 含义 | 本质 |
|------|------|------|
| 过程1：新生成不良 | 贷款从正常→不良（池1→池2） | **进水口**（核心原生指标） |
| 过程2：计提减值损失 | 利润注入减值准备（池4→池3） | **补水泵**（利润调节器） |
| 过程3：核销不良贷款 | 减值准备金冲销不良（池3→池2） | **出水口**（美容手术） |

在此基础上，**三维度交叉验证**防止单一指标被操纵：

1. **风险水平（静态水位）**：不良率、广义不良率（不良率+关注率）、不良偏离度（逾期90+/不良）、逾期90+占比 — 看清当下风险"水位面"
2. **风险迁徙（动态流速）**：正常类贷款迁徙率、次级/可疑类迁徙率 — 看清风险"流动方向和速度"
3. **风险抵补（防洪坝高度）**：拨备覆盖率、拨贷比、风险加权资产减值准备率 — 看清安全垫够不够厚

核心洞察：**不良率只是经核销修饰后的"快照"，新生成不良率才是原生指标**。只盯快照不看流速，等于只盯游泳池水位不看进水口和出水口各有多大 —— 水位没变，可能进水在加速。

## A1 — 书中的应用 (Past Application)

### 案例 1：招行 vs 兴业 — 不良率相近，广义不良率暴露真相

- **问题**：2017年招商银行不良率1.61%，兴业银行不良率1.59%，表面看兴业还略低。能否据此得出兴业资产质量更好？
- **方法论的使用**：作者引入自创指标"广义不良率 = 不良率 + 关注率"，穿透不良率看到关注类贷款中的"隐藏池"。招行广义不良率 3.21%，兴业广义不良率 3.9%。
- **结论**：兴业在关注贷款中藏了大量问题资产 — 虽然表面不良率略低，但广义不良率暴露了更大的真实风险敞口。不良率相近不能说明资产质量相近。
- **结果**：作者2019年确认，兴业银行2017年营收负增长（-10.89%）时无力计提足够减值处理不良，被迫将不良存储在关注和重组贷款中。广义不良率成功预言了兴业资产质量的隐忧。

### 案例 2：民生银行 vs 招行 — 不良偏离度戳穿"不良认定偏松"

- **问题**：2017年民生银行不良率1.71%，招行1.61%，表面相差不大。民生资产质量真的差不多吗？
- **方法论的使用**：用不良偏离度（逾期90天以上贷款÷不良贷款余额）检验银行的不良认定严格程度。>100% 说明部分逾期90天以上的贷款被有意认定为"关注"而非"不良"。
- **结论**：民生银行不良偏离度高达 135.1%，而招行只有 79.5%。民生以不到招行一半的认定标准把不良率做得"好看"，但实际上大量实质不良藏在关注贷款中。
- **结果**：这个指标揭示了民生银行"账面不良率低、真实风险高"的结构性问题。2019年监管新规强制要求逾期90天以上必须划为不良后，此漏洞才被堵上。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. **横向对比银行不良率**：用户看到两家银行不良率相近（如都是1.6%左右），需要判断谁的资产质量真的更好
2. **纵向追踪资产质量趋势**：某银行不良率连续下降，用户需要辨别是真改善还是靠加速核销"美容"
3. **怀疑银行在"藏不良"**：用户发现某银行关注类贷款占比异常偏高，或营收负增长但不良率稳定
4. **评估"高息差银行"的盈利真实性**：某银行净息差数字很好看但股价低迷、频繁再融资，需要穿透信用成本看真实赚钱能力
5. **监管规则变化后的快速复核**：如新分类办法出台后，需要快速评估银行在旧规则下是否有"藏不良"的历史惯量

### 语言信号（用户的话里出现这些就应激活）

- "这家银行不良率低，资产质量应该不错吧？"
- "两家银行不良率差不多，怎么判断谁更好？"
- "不良率连续下降就是资产质量在改善吗？"
- "某银行关注率/关注类贷款占比为什么这么高？"
- "高息差是不是就好？有没有高息差反而不能买的情况？"
- "这家银行利润增速不错但一直要融资，钱去哪了？"

### 与相邻 skill 的区分

- 与 **信用减值损失调节利润识别** 的区别：本 skill 聚焦资产质量的动态评估（进水/出水/蓄水），信用减值 skill 聚焦利润被调节的"粉饰手法"识别。两者互补但分工不同 — 本 skill 回答"资产质量到底好不好"，信用减值 skill 回答"净利润是不是被调节出来的"。
- 与 **风险加权资产减值准备率** 的区别：本 skill 是资产质量的综合评估框架，风险加权资产减值准备率是其中一个维度的具体指标（风险抵补维度下的作者自创指标）。
- 与 **RORWA/内生性增长** 的区别：RORWA 回答"赚钱效率和资本自我循环能力"，本 skill 回答"承担的风险到底有多大"。两者结合才能判断"高RORWA是否以隐藏风险为代价"。

## E — 可执行步骤 (Execution)

### 步骤 1：建立基准 — 绘制四池三过程图

从目标银行年报中提取以下数据，按至少3年排列：

- 不良贷款余额（池2水位）
- 贷款减值准备余额（池3水位）
- 信用减值损失（过程2：从池4→池3的注入量）
- 核销金额（过程3：从池3→池2的冲销量）
- 新生成不良率（过程1：若银行未披露，用下限公式估算）

**完成标准**：整理出一张3年×5项数据的时序表，肉眼可观测各池水位的动态变化。

**下限估算公式**（当新生成不良率未披露时）：
```
新生成不良率下限 = (不良余额增量 + 核销金额) ÷ 平均贷款总额
```

### 步骤 2：第一维度 — 评估风险水平（静态水位）

计算并对比以下四个指标的同业水平和自身趋势：

| 指标 | 计算 | 红色预警 |
|------|------|----------|
| 不良率 | 不良贷款÷贷款总额 | 单看绝对值意义有限 |
| 广义不良率 | 不良率 + 关注率 | 同业对比中持续偏高 |
| 不良偏离度 | 逾期90+贷款÷不良余额 | **>100%** 即不良认定偏松 |
| 逾期90+占比 | 逾期90+贷款÷贷款总额 | 与不良率差距持续扩大 |

**完成标准**：明确回答"这家银行当下的不良认定是严格还是偏松？关注类贷款中有多少潜在风险？"

**判停条件**：若不良偏离度 >100% 且广义不良率显著高于同业 → 直接判定该银行有"藏不良"嫌疑，进入步骤4综合判断时权重加倍。

### 步骤 3：第二、三维度 — 评估风险迁徙与风险抵补

**风险迁徙（动态流速）**：
- 追踪正常类贷款迁徙率（3年趋势）→ 上升 = 新问题贷款在加速涌入
- 追踪关注类贷款迁徙率 → 高企 = 关注类中蕴藏的"预备不良"在下迁
- 对比：不良率下降 + 正常类迁徙率上升 → 不良下降是核销的"假象"

**风险抵补（防洪坝）**：
- 拨备覆盖率趋势：持续下降需警惕（监管底线120%）
- 拨贷比趋势：注意和拨备覆盖率的三角关系（拨贷比 = 拨备覆盖率 × 不良率）
- 风险加权资产减值准备率：特别适用于非标/投资资产占比较高的银行（如兴业）

**完成标准**：回答两个核心问题 — (1) "资产质量在改善还是恶化？"(2) "现在的安全垫足够覆盖真实风险吗？"

### 步骤 4：交叉验证与"牌桌意图"推断

将步骤 1-3 的数据交叉对比，结合营收情况推断银行管理层的意图：

| 营收 | 不良暴露 | 拨备计提 | 推断 | 投资含义 |
|------|----------|----------|------|----------|
| 强 | 多暴露 | 多计提 | 主动清理、为未来储备余粮 | 利好 |
| 弱 | 少暴露 | 少计提 | 营收不够、被动隐藏 | ⚠️ 警惕 |
| 弱 | 少暴露 | 几乎不提 | 营收差+资产差、全线隐藏 | 🔴 危险 |

**关键判停条件**：若营收负增长但不良率稳定/下降 → 大概率存在"不良暴露意愿压制"→ 不宜仅凭不良率数字做乐观判断。应在营收恢复后预期不良"补暴露"。

### 步骤 5：输出综合结论

按以下模板输出：
1. **当前资产质量位态**：用一句话概括（如"不良认定严格、迁徙速度放缓、抵补充分"）
2. **关键风险点**：不超过3条（如"关注类占比偏高、新生成不良率下行速度慢于核销速度、拨备覆盖率处于下行通道"）
3. **管理层意图判断**：属于三种牌局中的哪一种
4. **跟踪关注点**：下一份年报应重点看什么指标验证判断

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **评估银行流动性风险**：本 skill 聚焦信用风险（贷款违约），不覆盖流动性风险（期限错配导致的资金链断裂）。流动性风险评估需要看同业负债占比、LCR、NSFR等指标
- **单一指标快速诊断**：如果有人希望"一个指标判断资产质量好坏"，告知不存在这样的指标，必须多维交叉验证
- **对非银行业企业的信用评估**：四池三过程模型专为银行业的拨备/核销机制设计，对一般企业不适用（一般企业用应收账款坏账计提模型）
- **银行估值定价**：本 skill 只产出"资产质量好不好"，不产出"值多少钱"。估值需结合 RORWA、内生性增长等 skill
- **仅比较同一年份的两家银行而不看趋势**：单年时点数据不足以判断银行资产质量方向，至少需3年纵向数据

### 作者在书中警告的失败模式

- **只看不良率就判断资产质量**（ce04）：不良率是经过核销修饰后的"快照余额"，关注类贷款是可操纵的"藏污纳垢之地"。民生银行不良率1.71%和招行1.61%表面相近，但不良偏离度分别为135.1%对79.5%，真实资产质量天差地别
- **高息差=好银行的陷阱**（ce09）：平安银行净息差名列前茅，但贷款信用成本高达2.39%~3.08%，算上信用成本后信贷业务基本赔钱。高息差来自"高利率覆盖高风险"模式，增收不增利，是"价值毁灭的典范"——证明不看信用成本的资产质量分析必然误判

### 作者的盲点 / 时代局限

- **新规后不良偏离度基本统一**：2019年4月《商业银行金融资产风险分类暂行办法》实施后，逾期90天以上强制划为不良，不良偏离度>100%的漏洞基本被堵上。但识别"不良暴露意愿"的方法论（营收弱时减暴露、营收强时增暴露的判断框架）不受影响
- **新生成不良率披露不统一**：目前仅招商银行和平安银行主动披露该指标，其他银行只能用下限公式估算，精度受限
- **非标资产的风险分类仍在过渡期**：新规将贷款五级分类扩展至全部信用资产，非标/投资资产的"不良"认定规则尚在完善中，本方法对投资资产占比高的银行（如兴业）的分析精度低于贷款占比高的银行

### 容易混淆的邻近方法论

- **与信用减值损失调节利润识别**：本 skill 从不良池水位的动态关系推断资产质量变化，"信用减值调节利润"从拨备前利润vs净利润增速的剪刀差推断利润粉饰。两者是同一枚硬币的两面，但分析入口不同
- **与拨备覆盖率/拨贷比的直观解读**：拨备覆盖率隐含"认可银行自己的不良认定"（不良是主观分母），拨贷比"完全不看风险差异"。两者各有极端，本 skill 在风险抵补维度同时使用三者（含风险加权资产减值准备率）交叉校准
