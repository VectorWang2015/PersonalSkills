# 财报全面分析

## 定位

聚合 skill，作为公司财报分析的唯一外部入口。按分析依赖关系自动编排 7 个原子 skill 的执行顺序，输出结构化综合报告。

## 输入要求

分析前必须确认：
1. **公司名称 + 报告期**（如"贵州茅台 2025Q1"）
2. **财报数据来源**（PDF/TXT 文件路径，或用户直接粘贴的关键数据）
3. 如果仅提供报表截图/摘要，提醒用户补充关键指标（ROE、毛利率、现金收入比率等）

## 执行流水线

按以下顺序逐层递进，每层完成后再进入下一层。每个子 skill 的详细方法论见 `atomic/<skill-slug>/SKILL.md`。

```
第一层：商业模式分类
└─ 1. 七步成诗法总框架 (atomic/qi-bu-cheng-shi-fa/SKILL.md)
       → 判断茅台型/沃尔玛型/银行型 → 确定后续分析侧重点
       → 确立横向(3-5家同行)+纵向(3-5年)对比基准

第二层：盈利地基（步骤1-4）
└─ 2. 盈利能力分析 (atomic/ying-li-neng-li-fen-xi/SKILL.md)
       → 扣非ROE → 现金收入比率 → 毛利率 → 净利润(净利率+现金含量+水分)

第三层：效率与安全（步骤5-6）
├─ 3. 运营效率分析 (atomic/yun-ying-xiao-lu-fen-xi/SKILL.md)
│      → 总资产周转率 → 资产结构扫描 → 存货/应收周转(2C/2B区分)
└─ 4. 杠杆评估 (atomic/gang-gan-ping-gu/SKILL.md)
       → 有息/无息负债区分 → 有息负债率 → 偿债安全边际 → 存贷双高检查

第四层：综合结论（步骤7）
└─ 5. 综合判断 (atomic/zong-he-pan-duan/SKILL.md)
       → ROE三维评判(数值/质量/稳定性) → 三好公司(行业/生意/数据)
       （依赖：第2-4步的全部数据）

第五层：风险控制（按需触发）
├─ 6. 避雷三阶段法 (atomic/bi-lei-fang-fa/SKILL.md)
│      → 仅当盈利能力或杠杆评估发现异常数据时执行
│      → 事前七筛 → 事中看大数字 → 事后三类分
└─ 7. 招股书快速评估 (atomic/zhao-gu-shu-ping-gu/SKILL.md)
       → 仅当分析对象为新股(上市<3年)或拟IPO公司时执行
```

## 输出格式

分析完成后，按以下结构输出综合报告：

```markdown
# <公司名称> <报告期> 全面分析

## 一、商业模式分类
### 公司类型判定: 茅台型 / 沃尔玛型 / 银行型
### 主要驱动因素 + 分析侧重点

## 二、盈利能力 (步骤1-4)
### 扣非ROE (表)
### 现金收入比率 (表)
### 毛利率 (横向对比表)
### 净利润三维检验 (表: 净利率/现金含量/扣非占比)
→ **盈利能力综合评级**: 优/良/差/红旗

## 三、运营效率 (步骤5)
### 总资产周转率 + 资产结构
### 关键周转指标 (存货/应收, 2C/2B判定)
→ **运营效率评级**: 高效/一般/低效/恶化

## 四、杠杆与安全 (步骤6)
### 有息/无息负债拆分 (表)
### 偿债安全边际 (现金/带息债务)
### 存贷双高检查
→ **杠杆安全评级**: 安全/可控/危险/红旗

## 五、综合判断 (步骤7)
### ROE三维评判: 数值/质量/稳定性
### 三好评估: 好行业 + 好生意 + 好财务数据
→ **最终结论**: 值得深入研究 / 暂放一边 / 建议排除

## 六、风险提示 (可选)
### 异常数据标注 + 避雷检查结果
```

## 保存分析结果

分析完成后，**必须**主动询问用户：

> "是否需要将以上分析保存为本地 Markdown 文件？"

- 若用户确认，保存到 `reports/<报告期>/` 目录
- 文件名格式：`<公司名称>-<报告期>-财报分析.md`
- 若用户拒绝，跳过

## 与原子 skill 的关系

本聚合 skill 编排所有原子 skill，原子 skill 不面向用户直接调用，不提供保存功能。所有入口和保存通过本 skill 统一管理。原子 skill 方法论文件位于 `atomic/` 子目录下，随本 skill 一起部署。
