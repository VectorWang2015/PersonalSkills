# 苹果树抽象模型

## R — 原文 (Reading)

> 人身险公司在春天卖出的保单，由于准备金计提和费用前置等原因，在今年冬天结账时很可能导致账面亏损。但在往后的岁月，由于保费不断流入，准备金计提与早期费用消失，如果公司不开展新的业务，净利润将迅速转正……这就好比种植一棵苹果树，在种树初期投入了较多的费用，但一旦树种下了，无论后续公司经营情况如何，都会有源源不断的"苹果"结出。
>
> — 东先生，《读懂保险股》第7章

> 一般会计准则的出发点是"从时间的起点去还原现在"……内含价值评估标准颇有"从时间的终点来还原现在"的味道。营运利润抱着在可接受度和真实度间寻找平衡的使命，在两者间极力斡旋。
>
> — 东先生，《读懂保险股》第7章

## I — 方法论骨架 (Interpretation)

人身险的商业模式有一个本质矛盾：**赚钱的时间点≠确认利润的时间点**。承保一张长期保单，首年产生大量费用（佣金、管理费、精算准备金），但保费收入是未来几十年的事。一般会计准则（GAAP）要求费用当期入账、收入分期确认，结果就是：**越努力经营（卖越多保单），账面越亏**。

苹果树模型把这个矛盾讲透了：

**种树 = 承保。** 挖坑、买苗、施肥——全是投入。反映在利润表上就是首年亏损（费用前置）。这笔"亏损"其实不是真亏，其中很大一部分以**剩余边际**的形式被强制锁入准备金——就像种子埋进土里，你看不见，但它值钱。

**结果 = 利润释放。** 树种下后，不需要每天再投大钱，苹果年年结。对应到保险：续期保费流入、费用已前置、准备金逐年摊销释放（剩余边际摊销）——利润从准备金蓄水池里"流"出来。

**三套准则 = 三个故事版本：**
- **GAAP 版本**（从过去还原现在）：只记录已发生的现金收支。结论——"这家公司年年亏损，种树越多亏得越惨，不行。"
- **内含价值版本**（从将来还原现在）：把未来所有苹果按11%贴现率折现回来算现在值多少钱。结论——"你不仅没亏，还有一大片会结果的苹果园，很值钱。"
- **营运利润版本**：去掉短期波动（投资波动、一次性事件），把剩余边际摊销+固定投资收益等稳定来源挑出来。结论——"别管账面那堆花活，每年稳定能赚这么多。"

**核心启示：种得越多亏得越多，但实际越赚钱。** 用GAAP利润来估值人身险公司，跟看见一个在种大面积苹果园的果农账本上是亏损的、就认为他不赚钱一样荒谬。

## A1 — 书中的应用 (Past Application)

### 案例 1: 四家公司"越增长越亏损"的报表幻觉

- **问题**: 2014-2019年间，中国平安、太保、国寿、新华四家公司披露的GAAP利润表表现各异。增速最快的公司（新单保费快速增长），其GAAP利润反而承压；而收缩新业务的公司，账面利润反而亮眼。
- **方法论的使用**: 作者运用苹果树模型解释这一"反直觉"现象：新单保费快速增长→费用前置激增→剩余边际进水口膨胀→利润表首年亏损扩大。反之，收缩新业务→费用骤降→仅有存量保单释放利润→账面利润反升。
- **结论**: 不能仅看GAAP利润的绝对数值或增速来判断保险公司经营好坏。需要结合新单增速、剩余边际变动、内含价值增长等多套准下的指标综合判断。
- **结果**: 确立了"GAAP只能用来排雷，不能用来估值"的铁律。一家保险公司在GAAP层面越差（增长越快导致费用越多），在内涵上可能越好。

### 案例 2: 中国平安2014-2019年剩余边际"蓄水池"运转

- **问题**: 平安寿险2019年剩余边际期末余额约9100亿，当期摊销约800亿（占比约8.8%）。投资者问：这个蓄水池是不是快漏完了？
- **方法论的使用**: 用苹果树模型解释蓄水池动态：进水口=新业务贡献的剩余边际（每年新签保单锁入）+预期利息增长（存量按5%复利）；出水口=摊销释放。只要进水>出水（新业务当年贡献的剩余边际 > 当期摊销），水池水位不降反涨。平安2019年新业务贡献的剩余边际约1000亿+预期利息增长约450亿，远超当期摊销800亿。
- **结论**: 不能孤立地看摊销比例，关键看"进水口-出水口"的剪刀差。只要新业务持续有价值注入，"水池"不会枯竭。
- **结果**: 建立"剩余边际不是一次性宝藏，而是持续更新的活水"的认知。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. **用户第一次接触保险股，对"为什么保险公司PE这么低"感到困惑** — 需要从根本上解释保险商业模式的特殊性
2. **用户在对比不同保险公司的利润表时，发现利润差异巨大，试图找出原因** — 需要用苹果树模型判断是经营差异还是商业模式差异
3. **用户看到卖方报告同时引用EV/PE/营运利润三个不同口径的"利润"，被绕晕了** — 需要用三套准则的三个故事版本来建立坐标系
4. **用户在判断保险公司增长质量时，不确定"保费增长30%但利润下降10%"是好是坏** — 需要识别"增长期亏损"vs"收缩期假繁荣"
5. **用户在做保险股估值前，需要先建立"什么才是保险公司的真正赚钱能力"的认知框架**

### 语言信号 (用户的话里出现这些就应激活)

- "为什么保险公司利润表亏损/PE这么低/看起来不赚钱？"
- "保险公司的GAAP利润能不能用？"
- "内含价值和会计利润到底什么关系？"
- "保险公司到底怎么赚钱的？"
- "为什么保费增长越快，利润反而越差？"
- "帮我理一下保险公司的商业模式"

### 与相邻 skill 的区分

- 与 `three-standards-worldview` 的区别: 苹果树模型是**理解三套准则关系的认知脚手架**（抽象比喻），而三套准则世界观框架是**三套准则各自的计算逻辑和钩稽关系的技术拆解**。苹果树解决"为什么"的问题，三套准则框架解决"怎么算"的问题。
- 与 `residual-margin-pool` 的区别: 苹果树模型用故事讲整体商业模式，剩余边际蓄水池模型是**剩余边际这一个具体机制的动态模拟**。苹果树是舞台，蓄水池是舞台上的一个道具。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **确认用户当前的困惑点在哪个层面**
   - 问："你更多是想理解 '保险公司为什么账面亏但实际赚' 这个商业模式悖论，还是想搞清楚 GAAP/EV/OP 三套指标之间的换算关系？"
   - 完成标准: 用户明确指出了困惑层面（商业模式本质 / 三套准则关系 / 具体估值歧义）。
   - 判停条件: 若用户问的是具体财务数据（如"平安2023年EV多少"），直接查询数据，跳过模型讲解。

2. **用苹果树故事建立直觉**
   - 用"种树-结果"类比解释费用前置/利润后置：
     - 种树=承保（投入费用→账面亏损→剩余边际入土）
     - 结果=摊销（费用已消化→利润释放）
   - 完成标准: 用户表示"明白了"或主动追问下一步，而非继续困惑同一个点。

3. **展开三个故事版本，建立坐标系**
   - GAAP 版本："从过去看现在"→只能排雷，不能估值
   - EV 版本："从未来看现在"→内含价值才是公司的真正价值
   - OP 版本："架桥"→剔波动看可预期利润
   - 完成标准: 用户能用自己的话说出三套准则分别"在看什么"。

4. **映射到用户的具体问题**
   - 将三个故事版本应用回用户最初提到的具体公司/数据
   - 完成标准: 用户的具体困惑被解答，或者用户转向新的、更深入的问题。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只是要查一个**具体的财务数据**（如"平安2023年EV"、"国寿NBV增长率"）— 这不需要认知框架，直接查数。
- 用户分析的是**财险公司**（如人保财险、平安产险）— 财产险的商业模式（短尾、一年期）不适用于费用前置利润后置的苹果树逻辑。
- 用户的问题**纯粹是数据计算**（如"按11%折现率，100亿10年后现值多少"）— 这是DCF基本功，不需要苹果树模型。
- 用户已经**充分理解三套准则关系**，需要的是具体的估值参数或操作指南 — 直接用内含价值拆解或估值方法 skill。

### 作者在书中警告的失败模式

- **"费用前置"导致的经营假象**：经营不善的公司（砍销售团队、停止新业务）反而短期账面利润大增，因为不再发生新单费用。投资者仅看当期利润表会得出"收缩期=好公司，扩张期=烂公司"的完全相反结论。（来源：ce12）
- **"不产生新业务也有增长"的价值陷阱**："即使保险公司不再产生新业务，内含价值也有xx%的增长"看似诱人，但新业务是价值增长的发动机，持续萎缩的新业务终将断流。存量苹果树终有枯竭之日。

### 作者的盲点 / 时代局限

- 苹果树模型默认"树一定能结苹果"，但这一前提在对假设极度敏感的内含价值框架中可能失稳：投资收益率假设微调50bp即可导致EV变动4-5%，且2016年四家上市公司曾集体下调假设导致EV一次性缩水。苹果树模型是一个**方向性的抽象模型**，不做定量精度，不能替代EV参数检验。
- 模型未涉及低利率/负利率环境下的极端情景：如果十年期国债跌至1%，5%的永续投资收益率假设能否站住脚？
- IFRS17新会计准则（2023年生效）改写了保险合同的收入确认方式，GAAP的"费用前置"程度可能有所缓解，苹果树模型的GAAP版本需要针对新准则做修正。

### 容易混淆的邻近方法论

- **DCF公理**：苹果树模型的"把未来苹果折现回来看现在值多少钱"本质是DCF，但苹果树是**保险专用比喻**，DCF是普适公理。不应直接用苹果树模型对非保险企业估值。
- **剩余边际蓄水池模型**：剩余边际是苹果树模型中"埋在土里的种子"，蓄水池模型讲种子的流入流出，苹果树讲整棵树的生长逻辑。讨论剩余边际的摊销速度、进水口增速等定量问题请切到剩余边际蓄水池模型。
