# 偿付能力体系及其对估值的影响

## R — 原文 (Reading)

> 偿付能力体系的核心内容，是评估保险公司在风险来临时，其对保单继续履约的能力，这点无论是偿付能力体系第一代，还是第二代，都从未改变。

> — 东先生，第4章

> 偿一代存在两个致命的缺点：一是分子实际资本的调整中，会计准备金转换为法定准备金的过程呆板僵滞，导致公司的实际资本过度低估，特别是对于以保障型业务为主的公司，蕴含在准备金里的巨大"宝藏"未得到认可；二是采用统一的量化标准，没有全方位考虑各保险公司的差异，导致只要规模大的公司看起来抗风险能力就强。

> — 东先生，第4章 "偿一代的缺陷"

> 对偿一代中暴露的两个问题，偿二代祭出了两大杀招：一是完善偿付能力体系，包括了定量分析、定性分析和市场监督三大支柱……二是修正保险公司实际资本与最低资本的计算，考虑各类业务的特点。偿二代的实施，标志着全面风险管控时代的来临。

> — 东先生，第4章 "巴菲特式的偿二代"

> 偿二代抛弃了呆板僵滞的法定准备金，期望企业采用"最优估计"评估负债端的情况……其中，改变最大的一点是蕴含在准备金中的巨大宝藏"剩余边际"不再被认为是认可负债的一部分，反而可以被认为是实际资本，由此，大幅拉抬了分子，减小了分母，使偿付能力充足率比此前大了许多。

> — 东先生，第4章 "杀招二：负债中准备金的评估方式"

> 按原保监会对外公布的数据，传统险占比高的寿险公司在偿二代下的偿付能力充足率预计会由 245% 提升至 311%，提升了约 27%。

> — 东先生，第4章

> 实际资本 = 认可资产 - 认可负债 + 1.4 × 过去12个月代理人渠道产生的期交保费

> — 引自东先生转引《中国偿二代C-ROSS体系探讨》（王晴）

> 在偿一代体系下，实际资本（分子）可由调节资产和负债得到，与净资产的差距虽然会有，但不至于很大（一般小于净资产 10%）；最低资本（分母）也可由公司的负债要素加权得到。在偿二代体系下……实际资本（分子）与净资产的关系已非直接关系，而仅仅是映射关系，这种映射关系表现为每1元钱的净资产对应远不止1元钱的实际资本。

> — 东先生，第4章

> 三个等式中的"持有偿付能力额度的成本"……都是指公司为了持有"最低资本"而付出的代价，这里的"最低资本"正是上一章讨论的偿二代下的"最低资本"。

> — 东先生，第5章

## I — 方法论骨架 (Interpretation)

### 偿付能力体系的核心命题

偿付能力体系的唯一核心问题：**保险公司在风险来临时，能否继续履行对保单持有人的赔偿责任？**

```
偿付能力充足率 = 实际资本（分子）÷ 最低资本（分母）
```

- **实际资本**：保险公司的"真金白银"——在监管口径下真正可用于应对风险的资本
- **最低资本**：监管认为"刚好够用"的资本门槛——充足率为100%时的边界值
- **监管红线**：偿付能力充足率 ≥ 100%，跌破即视为资本不足

### 偿一代：格雷厄姆式的量化主义

**机制**：效仿银行资本充足率，用统一的量化标准按保费/准备金规模加权计算最低资本，通过资产和负债的调节规则计算实际资本，两者一除得出偿付能力充足率。

| 维度 | 偿一代做法 | 效果 |
|------|-----------|------|
| 实际资本（分子） | 认可资产 - 认可负债。负债端：会计准备金→法定准备金（采用保守评估利率） | 剩余边际被强制计入负债，分子被压低 |
| 最低资本（分母） | 按保费/准备金规模的固定比例加权（如保险合同负债×4%） | 规模越大→分母越大，奖粗放扩张 |
| 基本逻辑 | 纯量化主义，"有罪推定"——先假设风险高，再用数字证明没问题 | 格雷厄姆式 |

**两大致命缺陷**：

1. **分子被系统性低估**：法定准备金转化过程呆板僵滞，保障型公司准备金里的"宝藏"（剩余边际）被当成负债而非资本
2. **分母只看规模不看质量**：两个体重相同的人——一个啤酒肚、一个满身肌肉——在偿一代眼中"同一个健康水平"

### 偿二代（C-ROSS）：巴菲特式的全面风险管理

偿二代从纯量化走向"定量+定性+市场监督"三位一体，标志着全面风险管控时代的来临。

#### 杀招一：建立三大支柱

```
          ┌──────────────┐
          │  三大支柱     │
          ├──────────────┤
          │ 第一支柱：定量 │ ← 偿付能力充足率（偿一代仅此一项）
          │ 第二支柱：定性 │ ← SARMRA评估（风险管理要求与评估）
          │ 第三支柱：市场监督│ ← 公开披露（投资者可查偿付能力报告摘要）
          └──────────────┘
                ↑
         互相纠缠、互相引用
```

| 支柱 | 核心内容 | 投资者的使用方式 |
|------|----------|-----------------|
| **定量分析** | 偿付能力充足率计算（实际资本÷最低资本） | 核心关注指标，参与内含价值的持偿成本计算 |
| **定性分析** | SARMRA评估：监管每年现场评估+集中答辩。评分低者被要求更高的充足率 | 评分高低间接影响最低资本数值 |
| **市场监督** | 公司须定期在网站上披露偿付能力报告摘要 | 直接查阅各公司偿付能力状况，无需依赖主观判断 |

#### 杀招二：修正实际资本与最低资本的计算

**（A）实际资本（分子）的质变：剩余边际被认可为资本**

这是偿付能力体系变革中最具划时代意义的一项：

```
偿一代：剩余边际 ∈ 认可负债 → 压低分子 → 压低充足率
偿二代：剩余边际 ∈ 实际资本 → 拉抬分子 → 拉抬充足率
```

- 偿二代抛弃了保守的法定准备金，采用"最优估计"评估负债
- **剩余边际不再是负债，而是资本**——这对以保障型产品为主的公司是巨大利好
- 近似理解公式：**实际资本 ≈ 认可资产 - 认可负债 + 1.4 × 过去12个月代理人渠道期交保费**

**（B）最低资本（分母）的精细化：从规模导向到风险导向**

| 项目 | 偿一代 | 偿二代 |
|------|--------|--------|
| 计算方式 | 保费/准备金×固定比例 | 市场风险 + 信用风险 + 保险风险（偏导数） |
| 考虑因素 | 仅规模 | 业务质量、风险类型、SARMRA评分调节 |
| 对保障型公司 | 分母偏高（不区分业务质量） | 分母增幅 < 分子增幅 → 充足率大幅上升 |
| 对短期理财型公司 | 分母正常 | 分母大幅上升 → 充足率下降 |

**结论**：偿二代是一个"奖励质量、惩罚规模"的制度设计。保障型公司偿付能力充足率从~245%→~311%（+27%），短期理财型公司则遭遇滑铁卢。

### 偿付能力体系与内含价值的桥梁：持偿成本

**最关键**：偿付能力充足率不是孤立指标，它通过**持偿成本**桥接到内含价值体系：

```
内含价值 = 调整净资产 + 有效业务价值 + 持偿成本（负数）
扣偿后新业务价值 = 新业务价值 + 归属于新业务的持偿成本（负数）
```

持偿成本的计算逻辑（"时空旅程折返"）：

```
最低资本 → 按5%投资收益率推到保单生命终点 → 按11%贴现率贴回现在 → 净折损 = 持偿成本
```

| 维度 | 含义 |
|------|------|
| **为什么是成本？** | 最低资本虽然可以投资（赚5%），但不能分红、不能自由使用。用高贴现率（11%）折现未来的5%低回报，中间6%的"利差损"就是持有代价 |
| **对谁影响大？** | 最低资本越大→持偿成本越大→内含价值越低。保障型公司在偿二代下最低资本降低→持偿成本降低→内含价值上升 |
| **新业务vs存量** | 新保单的持偿成本占比（约15-16%）高于存量保单（约7%），因为新单需一次性锁定额外最低资本 |

## A1 — 书中的应用 (Past Application)

### 案例 1：偿二代切换 — 中国平安调整净资产暴增41%

- **问题**：2016年偿一代→偿二代的切换如何影响内含价值？
- **方法论的使用**：作者以平安2016年年报数据为素材。偿一代下调整净资产3279亿→偿二代下3482亿，寿险部分偿一代1019亿→偿二代1222亿，暴增19.9%（41%疑为寿险部分某口径）。核心原因：偿二代下内含价值准备金弃用保守评估利率，采用最优估计折现率→负债下调→调整净资产上升。
- **结论**：偿二代让保障型保险公司摆脱"保守而刻板"的法定准备金束缚。剩余边际被认可为实际资本，实现"轻资产"商业模式。调整净资产增幅反映的是被旧准则禁锢的真实价值释放。
- **结果**：作者强调——理解偿二代的内含价值逻辑，是看懂保险股真实价值的前提。若不理解这一制度切换，投资者看到的"调整净资产暴增"可能会误读为经营改善，实则是制度红利的一次性释放。

### 案例 2：新华保险2015年 — 三套标准下的净资产"罗生门"

- **问题**：同一家公司在不同偿付能力标准下，净资产到底是多少？
- **方法论的使用**：新华保险2015年在三套计量标准下呈现三种净资产：
  - 会计净资产：578亿（最低）
  - 偿一代调整净资产：500亿（最保守，比会计口径还低）
  - 偿二代调整净资产：768亿（最高，比会计口径+33%，比偿一代+54%）
- **结论**：189亿差额本质上是被偿一代"禁锢"的保险公司真实价值——剩余边际。偿一代不仅不如一般会计准则宽容，甚至将剩余边际进一步压低。投资保险股必须回答的首要问题："你看的是哪个世界的净资产？"
- **结果**：这个案例是作者论证"三套准则世界观"的经典素材——同一实体在不同制度透镜下折射出截然不同的净资产数值，单一标准必然导致错误判断。

### 案例 3：偿二代对不同业务类型公司的差异化影响

- **问题**：同样是保险公司，为什么有些公司在偿二代下受益巨大，有些却受损？
- **方法论的使用**：作者通过对比机制阐明了偿二代的分配效应：
  - **受益者**：以长期保障型业务为主的公司（如平安、太保转型后）
    - 分子（实际资本）：剩余边际被认可→大幅拉抬
    - 分母（最低资本）：保障型保单风险权重低→增幅慢于分子
    - 净效果：偿付能力充足率从~245%→~311%
  - **受损者**：以短期理财型/投资型业务为主的公司
    - 分子：剩余边际本就少，被认可为资本的红利几乎吃不到
    - 分母：短期险风险权重高→分母大幅上升
    - 净效果：偿付能力充足率下降，扩张受限
- **结论**：偿二代是一把"结构性手术刀"——它用制度设计倒逼行业从规模导向转型为价值导向。看懂偿付能力体系的进化方向，就能预判监管激励的方向。
- **结果**：这一案例构成了后续理解"为什么保障型公司估值更高"的制度基础。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill？

1. **理解偿付能力充足率变动的含义**：用户看到某公司偿付能力充足率大幅上升或下降，需要理解是经营改善还是制度红利释放
2. **评估偿二代二期（2022年）的影响**：用户需要理解偿二代二期收紧实际资本认定后，哪些公司偿付能力充足率会显著下降，进而影响内含价值
3. **分析持偿成本对估值的拖累**：用户问"最低资本到底拖累了多少内含价值"，需要建立偿付能力→持偿成本→内含价值的量化传导链条
4. **对比保险公司的"资本效率"**：两家偿付能力充足率相近的公司，实际业务结构可能截然不同，需要穿透充足率数字看资本质量
5. **理解监管政策方向对投资的影响**：用户想预判"监管是鼓励保障型还是理财型"，通过偿付能力体系的制度激励方向来推断
6. **剩余边际双重身份的理解**：用户困惑于"剩余边际到底是负债还是资本"，需要解释它在GAAP和偿二代下的双重身份

### 语言信号

- "偿二代是什么？和偿一代有什么区别？"
- "偿付能力充足率怎么算的？"
- "为什么剩余边际在偿二代下变成资本了？"
- "最低资本和持偿成本之间什么关系？"
- "偿二代二期对保险公司有什么影响？"

- "SARMRA评估是什么？"

### 与相邻 skill 的区分

- 与 **ev-valuation**（内含价值估值）：偿付能力体系是内含价值计算的前置条件——最低资本决定持偿成本，持偿成本是内含价值三大构件之一。本 skill 解释"为什么有这个构件"，ev-valuation 解释"这个构件怎么参与估值计算"。两者构成"制度基础→估值应用"的递进关系。
- 与 **residual-margin-analysis**（剩余边际分析）：剩余边际在GAAP下是准备金里的"利润暂存库"，在偿二代下被认可为实际资本。本 skill 解释剩余边际在监管维度的"制度身份"，剩余边际分析 skill 解释其在会计维度的"蓄水池动态"。
- 与 **triple-framework**（三套准则框架）：三套准则框架包含GAAP/EV/OP三个世界观，本 skill 聚焦第四个维度——监管准则（偿付能力体系），它是三套准则框架的"制度底层"。E、OP 三者的数量关系都受到偿付能力体系的约束。

## E — 可执行步骤 (Execution)

### 步骤 1：提取偿付能力核心指标 — 建立制度基准线

从目标保险公司年报"偿付能力报告摘要"或"风险管理"章节提取以下数据（至少2期）：

| 指标 | 来源位置 | 说明 |
|------|----------|------|
| 综合偿付能力充足率 | 偿付能力报告摘要 | 监管核心指标，≥100%为合格 |
| 核心偿付能力充足率 | 同上 | 仅含核心资本，更严格的标准 |
| 实际资本（总额） | 偿付能力报告 | 分子，偿二代下远大于会计净资产 |
| 其中：核心一级资本 | 偿付能力报告 | 最高质量的资本 |
| 最低资本（总额） | 偿付能力报告 | 分母，含市场风险/信用风险/保险风险 |
| 最近一期风险综合评级 | 偿付能力报告 | A/B/C/D级，A为最优 |
| SARMRA评分 | 偿付能力报告/风险管理章节 | 定性评估得分，影响最低资本计算 |
| 会计净资产（合并口径） | 资产负债表 | 用于计算"实际资本/净资产"倍数 |

完成标准：计算"实际资本 ÷ 会计净资产"，若 >1.5 说明剩余边际被大量认可为资本（保障型公司特征），若 ≈1.1 说明公司以短期/理财型业务为主。

### 步骤 2：诊断偿付能力的"含金量" — 区分制度红利与真实安全

偿付能力充足率自身不足以判断安全，需拆解其构成质量：

| 判断维度 | 信号 | 含义 |
|----------|------|------|
| 实际资本/净资产 | >1.5x | 剩余边际大量释放为资本→保障型业务占比高 |
| 实际资本/净资产 | ≈1.0-1.1x | 剩余边际贡献小→理财型/短期险为主 |
| 充足率 vs 同业 | 显著高于同业 | 可能是业务结构导致→分析结构；可能是资本闲置→效率损失 |
| 充足率 vs 同业 | 逼近100%红线 | 危险信号，需立即查看是否由以下驱动：①新业务消耗资本过快 ②投资亏损侵蚀净资产 ③实际资本认定被收紧 |
| 核心 vs 综合差距 | 差距很大 | 存在大量附属资本（质量较低），需关注附属资本构成 |
| 风险综合评级 | A→B→C趋势 | 经营风险在恶化，即使充足率数字尚可 |

判停条件：若偿付能力充足率 < 120% 且持续下降 → 公司接近监管红线，可能面临：限制新业务、限制分红、要求股东增资。这是重大的基本面恶化信号，应立即提醒用户。

### 步骤 3：从偿付能力桥接到内含价值 — 理解持偿成本的影响

持偿成本无法直接从偿付能力报告中读出，但可以通过结构分析推断其方向：

```
偿二代利好长期保障型公司 → 最低资本（分母）增幅 < 实际资本（分子）增幅
                          → 签发同量新单所需额外最低资本少
                          → 持偿成本占比低
                          → 扣偿后新业务价值更高
                          → 内含价值更真实
```

关键检查点：

| 检查项 | 怎么做 | 结论 |
|--------|--------|------|
| 新单的持偿成本占比 | 从年报内含价值章查"持偿成本÷扣偿前新业务价值" | 正常约15-16%，保障型公司偏低 |
| 存量有效业务持偿成本占比 | "持偿成本÷扣偿前有效业务价值" | 正常约7%，远低于新业务（因资本在保单后期递减） |
| 持偿成本占比趋势 | 连续3年纵向对比 | 下降 → 业务结构向保障型转型；上升 → 短期险增多或资本成本上升 |

完成标准：能回答"这家公司签发新保单需要锁定多少额外最低资本？这种资本锁定在行业中是高还是低？"

### 步骤 4：评估偿二代二期（2022年实施）的影响

偿二代二期对本书内容做了重要修正，需关注：

| 变化维度 | 偿一代→偿二代一期 | 偿二期追加 | 受影响最大的公司 |
|----------|-------------------|-----------|----------------|
| 实际资本认定 | 剩余边际被认可 | 对剩余边际进行分级（核心/附属），部分不计入核心资本 | 剩余边际占净资产比重极高的保障型公司 |
| 最低资本计算 | 考虑风险类型差异化 | 进一步细化风险因子，增加集中度风险等新风险模块 | 资产配置集中的公司 |
| 利率风险资本 | 引入利率风险计量 | 强化利率下行风险的最低资本要求 | 固收配置比例极高的公司（全员） |

操作建议：若分析的是2022年以后的数据，直接用年报披露的偿二代二期口径数据。若对比2022年前后的历史数据，需要提醒用户口径不可直接比较。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- **仅询问内含价值数字或计算公式**：偿付能力体系通过持偿成本影响内含价值，但若用户只问EV多少/怎么算，应转至 ev-valuation skill。本 skill 的触发条件是用户明确表达对偿付能力制度本身的困惑。
- **产险公司偿付能力分析**：产险的偿付能力逻辑、最低资本结构与人寿险完全不同（产险保单多为一年期，无剩余边际概念）。本 skill 仅适用于人身险公司。
- **银行资本充足率的对比**：偿付能力充足率和银行资本充足率虽形似（都是"资本÷风险加权资产"），但实际资本的含义截然不同（保险的剩余边际无银行对应物）。直接类比会导致严重误判。
- **偿付能力充足率的单一数字查询**：如仅问"平安偿付能力充足率多少"，直接用数字回答即可，无需激活整套分析框架。
- **非中国保险公司的偿付能力分析**：本框架基于中国 C-ROSS，欧盟 Solvency II、美国 RBC 等体系的制度逻辑和参数均不同，不可直接套用。

### 作者在书中警告的失败模式

- **直接用偿一代和偿二代的充足率数值做历史对比（ce04）**：偿一代→偿二代的切换是一次性制度红利释放，不可将2016年前后的充足率数值视为经营趋势变化。正如平安调整净资产"暴增41%"是制度切换所致，非经营改善。
- **偿一代下的"规模即安全"幻觉（ce04延续）**：在偿一代体系下，规模越大的公司偿付能力充足率越高。这种制度已废止，但投资者可能仍潜意识里将"保费规模大"等同于"偿付能力安全"——这是偿一代的制度惯性误导。
- **只看偿付能力充足率不看业务结构（ce06延伸）**：偿二代下，两家充足率相同的公司，一家以长期保障型为主、另一家以短期理财型为主，前者实际面临的风险远低于后者。因为保障型的剩余边际提供了资本缓冲，而理财型面临利差损和流动性双重风险。充足率数字"平均"了结构差异。
- **新华人寿2004年偿付能力爆雷事件（ce03）**：2004年新华保险因偿付能力充足率严重不足+管理层腐败，被保险保障基金接管。这是作者反复引用的警戒案例——偿付能力充足率不是可有可无的合规指标，跌破红线意味着"生存权"受到威胁。投资者必须在持有保险股的过程中，持续跟踪这一指标。

### 作者的盲点 / 时代局限

- **偿二代二期（2022年实施）未覆盖**：本书基于偿二代一期（2016年实施）。2022年实施的偿二代二期对以下方面做了重大修订：①剩余边际被分级，部分不再计入核心资本；②利率风险最低资本大幅提高（对固收配置比例高的保险全员收紧）；③增加了集中度风险等新风险模块。这些修订可能显著拉低部分保险公司的偿付能力充足率，进而增加持偿成本，降低内含价值。本 skill 的分析框架不变，但参数需要用户结合最新年报数据更新。
- **低利率环境对最低资本计算的新增压力**：本书写作时十年期国债约3.1%，偿二代利率风险的最低资本计量相对宽松。截至2025年，十年期国债降至约1.6-1.7%，利率下降本身会增加利率风险最低资本（利差损风险暴露加大），进一步推升最低资本→增加持偿成本→压低内含价值。这一传导机制在低利率环境下会被放大。
- **作者以2019年平安数据为标杆，中小公司偿付能力的"隐蔽问题"被低估**：平安、国寿等头部公司的偿付能力充足率长期保持在250%以上，但许多中小人身险公司（尤其是以万能险为主的）偿二代下充足率仅略高于100%红线。本书未系统讨论中小公司的偿付能力脆弱性，这部分"尾部风险"容易被头部数据的"行业平均"所掩盖。
- **未覆盖偿付能力与IFRS17的互动影响**：IFRS17（2023年实施）改变了保险合同负债的计量方式，可能间接影响认可负债的认定→影响实际资本。两套新制度（偿二代二期+IFRS17）的同时实施产生了叠加效应，本书成书时两者均未落地。

### 容易混淆的邻近方法论

- **偿付能力充足率 vs 银行资本充足率**：形似神异。银行的实际资本≈净资产±调整，保险的实际资本=净资产+剩余边际等，两者的"水分"完全不同。保险的偿付能力充足率250%≠银行的CAR 25%更安全，不可横向比较。
- **偿付能力充足率 vs 保险公司的净资产/总资产（杠杆率）**：偿付能力充足率的分母是"最低资本"（风险导向），而非总资产（规模导向）。两家资产规模相同的公司，保障型多的"最低资本"小，偿付能力充足率反而高。这是设计意图——不是bug。
- **偿二代 vs Solvency II（欧盟）**：两者都是三大支柱结构，但参数和侧重点不同。Solvency II更偏重市场一致性估值，C-ROSS保留了中国特色（如评估利率的监管指导）。不可用欧盟的偿付能力逻辑套中国保险公司。
- **核心偿付能力充足率 vs 综合偿付能力充足率**：核心只含核心资本（质量最高），综合含附属资本（补充资本）。两者差距越大，资本质量越低。类似银行的"核心一级资本充足率"和"资本充足率"的区别。

## 相关 skills (阶段 3 填充)

- **prerequisite-for**: `ev-valuation` — 偿付能力体系中的"最低资本"直接决定内含价值里的"持偿成本"。理解偿付能力是理解内含价值估值的前提。若不理解最低资本为何存在、如何计算，就无法理解持偿成本为什么是必须扣除项。
- **composes-with**: `residual-margin-analysis` — 剩余边际在偿付能力体系下被认可为实际资本（监管维度），在GAAP下属于准备金（会计维度）。两 skill 从不同角度描述同一个科目的双重身份——前者回答"制度为何允许它成为资本"，后者回答"经济实质为何它是利润"。
- **contrasts-with**: `triple-framework` — 三套准则框架（GAAP/EV/OP）是商业估值维度的坐标系，偿付能力体系是监管约束维度的坐标系。前者回答"公司值多少钱"，后者回答"公司能开多大的马力"。两者在持偿成本处交汇。
